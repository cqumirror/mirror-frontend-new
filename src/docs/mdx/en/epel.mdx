## EPEL Mirror Usage Guide

## URL

https://mirrors.cqu.edu.cn/epel

## Description

EPEL is a collection of commonly used packages for Red Hat-based distributions (such as Rocky Linux, CentOS, Fedora, RHEL, etc.), maintained by Fedora.

## Supported Architectures

- x86_64
- aarch64

## Supported Versions

`release` and `testing`

## Usage

If a configured epel repo already exists, back it up first:

```bash
# fedora
mkdir epel-backup
mv /etc/yum.repos.d/epel*.repo /etc/yum.repos.d/epel-backup/.
```

### Online

When online:


```bash
# Install epel-release
dnf install epel-release
```

Configure the repo file:

```bash
# fedora epel
for i in {'',-testing}; do wget https://mirrors.cqu.edu.cn/repo/epel/epel${i}.repo; done
```

Then run:

```bash
sudo dnf makecache
```

to generate the cache.

### Offline

After confirming that no dependencies are missing, run:

```bash
# fedora epel
for i in {'',-testing}; do wget https://mirrors.cqu.edu.cn/repo/epel/epel${i}.repo; done
```

Then run:

```bash
dnf install epel-release
```

to add the package signature and related content.
If this step fails, manually obtain the `epel-release` file from the `epel` repository and use `rpm -i <file>.rpm` to install it. Then follow the steps above to replace the repo file.
