## Alpine Linux Mirror Usage Guide

## Address

https://mirrors.cqu.edu.cn/alpine

## Description

Alpine Linux package repository. Alpine is a lightweight distribution that uses busybox and musl libc.

## Supported Architectures

x86_64

## Supported Versions

- `latest-release`
- `edge`

## Usage

Alpine has three repositories:

- **main** contains software directly supported and maintained by the Alpine core and primary teams. It also has official special documentation and is always available for all releases. If certain packages are no longer maintained upstream, alternatives will be provided. Packages are generally chosen for their upstream responsibility and stability. Packages from community (or rarely testing) are accepted and moved to the main repository.
- **community** contains packages made by users in collaboration with official developers, following the Alpine package workflow. They are supported by these contributors and may stop being updated if the contributors leave; they may also be removed in future releases due to lack of upstream support. Accepted testing packages are moved to the community repository.
- **testing** is where new packages live. These are made by any Alpine community contributor. Testing is not linked to any release. The testing repository is only available in the `edge` version. Accepted testing packages will be moved to the community repository or (rarely) the main repository.

The package repositories used for retrieving installation files are specified in the file `/etc/apk/repositories`. Each line in this file specifies the location of a package repository and an optional label. The location can be a URL or a path to a directory on the local filesystem. Labeled repositories are prefixed with the specifier `@tag`, followed by a space and the repository location.

The default looks like:

```txt
#/media/cdrom/apks
http://dl-cdn.alpinelinux.org/alpine/v3.18/main
http://dl-cdn.alpinelinux.org/alpine/v3.18/community
```

Switch to the Chongqing University open-source mirror:

```bash
sed -i 's|dl-cdn.alpinelinux.org|mirrors.cqu.edu.cn|g' /etc/apk/repositories
```

If you want to add the testing repository, add the following line, replacing `<release-ver>` with the actual value:

```bash
http://mirrors.cqu.edu.cn/apline/<release-ver>/testing
```

Then run:

```bash
apk update
```

to update the index.
